# Инструкция к запуску
Запуск проекта осуществляется через docker при помощи docker compose up -d --build

При первом запуске необходимо в корне проекта создать файл .env и дополнить информацию о базе данных, пример его наполнения находится в .env.example.

Также необходимо создать .env внутри backend/laravel/ и также дополнить информацию о  базе данных/телеграм боте, пример наполнения находится в .env.example по пути backend/laravel/.

После создания конфигурационных файлов .env, требуется выполнить docker compose exec php composer install, для загрузки требуемых библиотек.

После установки требуемых библиотек следует выполнить docker compose exec php php artisan key:generate, для генерации ключа приложения.

Далее необходимо выполнить docker compose exec php php artisan migrate для выполнения миграций в базу данных.

После выполнения миграций необходимо выполнить docker compose exec php php artisan migrate --seed для наполнения базы данных тестовыми данными через seeder, можно указать свои данные в backend/laravel/database/seeders/ModuleSeeder.php для быстрого наполнения базы данных.

При изменении клиентской части (frontend'a) необходимо выполнить docker compose up -d --build nginx для пересборки клиентской части и применения изменений.

Для удобной работы с базой данных в контейнере находится phpmyadmin - графический интерфейс с помощью которого можно быстро изменять данные внутри базы данных.

Чтобы открыть страницу проекта необходимо перейти по http://localhost:80

Для открытия phpMyAdmin необходимо перейти по http://localhost:8081 

# Тестирование 
При переходе по http://localhost:80 должна отображаться страница лендинга.

При нажатии на кнопки Оставить заявку/Получить демо страница должна прокручиваться вниз к форме.

При заполнении формы валидными данными, а именно: Имя должно состоять из латинницы, либо кириллицы, может содержать тире. Номер телефона должен состоять только из цифры, не менее 10 и не более 12. Электронная почта должна состоять только из латинницы, содержать @, также может содержать другие спец.символы. Пример почты - example@mail.com.

При отправке формы с валидными данными появится сообщение об успешной отправке данных и данные отобразятся в чате с телеграм ботом(если был подключен), а также сохранятся в базе данных.

При отправке формы с невалидными данным, поле непрошедшее валидацию будет подсвечено красным и появится надпись указывающая на ошибку.
